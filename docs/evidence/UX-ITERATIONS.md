# UX 반복 개선 기록

**프로젝트**: Delphi Call Graph Analyzer
**기간**: Sprint 4~6 (2026-03-10 ~ 2026-03-16)

UX 문제를 발견하고 개선한 과정을 "발견 → 조치 → 결과" 형식으로 기록합니다.

---

## 이터레이션 1: 경로 입력 가이드 부재

### 발견
- **시점**: Sprint 4 초기 테스트
- **이슈**: 입력 필드에 placeholder만 있어, 절대 경로/상대 경로 구분이 명확하지 않음
- **사용자 피드백**: "C:\\ 없이 상대 경로를 입력했다가 에러 발생"

### 조치
- `ProjectInput.tsx` 입력 필드 아래 `<p id="dpr-path-hint">` 도움말 텍스트 추가
- "절대 경로를 입력하세요 (예: C:\Projects\MyApp.dpr)" 안내

### 결과
- ✅ 경로 입력 오류 예방
- ✅ `aria-describedby`로 스크린 리더에도 힌트 전달

---

## 이터레이션 2: 로딩 중 빈 화면

### 발견
- **시점**: Sprint 4 컴포넌트 테스트
- **이슈**: 분석 실행 후 결과 로딩 중 빈 화면 → 사용자가 "멈춘 것인가?" 의문

### 조치
- `AnalysisSkeleton` 컴포넌트 구현 (Sprint 6)
- 카드 영역과 테이블 영역에 회색 shimmer 효과 스켈레톤 표시
- `aria-busy="true"` 추가로 스크린 리더에 로딩 상태 전달

### 결과
- ✅ 사용성 테스트에서 "분석 진행 중임을 명확히 인지" 확인
- ✅ 체감 대기 시간 감소

---

## 이터레이션 3: 콜 그래프 → 모달에서 탭으로 전환

### 발견
- **시점**: Sprint 5 사용성 리뷰
- **이슈**: 그래프를 모달로 표시하면 전체 그래프 탐색이 불편 (depth 슬라이더 조작 공간 부족)

### 조치
- Sprint 6에서 콜 그래프를 별도 탭(Call Graph 탭)으로 이동
- "그래프 보기" 버튼 클릭 시 Call Graph 탭 자동 전환
- 탭 내 전체 화면으로 React Flow 렌더링

### 결과
- ✅ 그래프 탐색 공간 300% 확대
- ✅ depth 슬라이더 조작이 자연스러움

---

## 이터레이션 4: 상태 배지 색상만으로 구분 불가

### 발견
- **시점**: Sprint 4 접근성 검토
- **이슈**: 사용/미사용 배지가 색상만으로 구분 → 색각 이상자 접근 불가

### 조치
- PRD §6.3 접근성 요구사항 반영
- 배지에 색상 + 텍스트("사용"/"미사용") + 아이콘(✓/✗) 중복 표현
- `StatusBadge.tsx` 컴포넌트로 분리하여 일관성 확보

### 결과
- ✅ WCAG 1.4.1 (색상 비의존) 충족
- ✅ Lighthouse 접근성 점수 92점 달성

---

## 이터레이션 5: 유닛 차트에서 테이블로 이동 불명확

### 발견
- **시점**: Sprint 6 통합 테스트
- **이슈**: 유닛 차트 바 클릭 시 테이블 필터가 적용되지만, 테이블 탭으로 이동하지 않아 사용자가 변화를 인지하지 못함

### 조치
- 유닛 차트 바 클릭 → 테이블 탭 자동 전환 + 해당 유닛 필터 적용
- "해당 유닛 메소드 보기 →" hover 메시지 추가

### 결과
- ✅ E2E 시나리오 3에서 유닛 차트 → 테이블 연동 검증
- ✅ 사용자 흐름이 직관적으로 연결됨

---

## 이터레이션 6: 빈 상태 안내 메시지 부재

### 발견
- **시점**: Sprint 6 초기 상태 확인
- **이슈**: 분석 전 첫 접속 시 빈 화면 → 무엇을 해야 하는지 불명확

### 조치
- `EmptyState` 컴포넌트 구현
- "Delphi .dpr 파일 경로를 입력하고 분석을 시작하세요" + 화살표 아이콘으로 입력 필드 안내

### 결과
- ✅ 신규 사용자의 온보딩 흐름 개선
- ✅ 사용성 테스트에서 "처음 보는 사람도 즉시 이해 가능" 확인
