## 테스트 시나리오 1: 첫 분석 실행

**목표**: 새 사용자가 .dpr 파일을 분석하고 결과를 확인할 수 있는가?

### Task 흐름
1. 브라우저에서 `http://localhost:5173` 접속
2. 입력 필드에 `.dpr` 경로 입력
3. "분석 시작" 버튼 클릭
4. 분석 결과(요약 카드 + 메소드 테이블) 확인

### 결과

| 단계 | 상태 | 메모 |
|------|------|------|
| 초기 화면 로드 | ✅ | 깔끔한 입력 폼 표시 |
| 경로 입력 | ✅ | placeholder로 예시 경로 제공 |
| 분석 버튼 클릭 | ✅ | 로딩 스피너 즉시 표시 |
| 요약 카드 렌더링 | ✅ | 4개 카드 (전체/사용/미사용/비율) |
| 메소드 테이블 렌더링 | ✅ | 컬럼 정렬, 상태 배지 표시 |

**완료 시간**: 약 8초 (분석 < 1초 + UI 렌더링)
**클릭 수**: 2회 (입력 + 버튼)

### 발견된 이슈
- **경미**: 절대 경로 vs 상대 경로 가이드 미표시 → placeholder 텍스트로 해소
- **없음**: 주요 UX 장애 없음

## 테스트 시나리오 2: 미사용 메소드 찾기

**목표**: 개발자가 프로젝트에서 사용되지 않는 메소드를 빠르게 식별할 수 있는가?

### Task 흐름
1. 분석 완료 후 메소드 테이블에서 "미사용" 필터 선택
2. 미사용 메소드 클릭으로 상세 패널 열기
3. callers/callees 정보 확인

### 결과

| 단계 | 상태 | 메모 |
|------|------|------|
| "미사용" 필터 적용 | ✅ | 드롭다운에서 즉시 필터링 |
| 미사용 배지 시각화 | ✅ | 빨간 pill 배지, 색상+텍스트 중복 표현 |
| 메소드 행 클릭 | ✅ | 슬라이드 패널 애니메이션으로 열림 |
| callers 목록 표시 | ✅ | "이 메소드를 호출하는 곳 없음" 명시 |
| callees 목록 표시 | ✅ | 이 메소드가 호출하는 메소드 목록 |
| 패널 닫기 (×, ESC) | ✅ | 두 방법 모두 동작 |

**완료 시간**: 약 15초 (필터 + 클릭 + 확인)
**클릭 수**: 3회 (필터 선택 + 행 클릭 + 패널 닫기)

### 발견된 이슈
- **없음**: 필터와 상세 패널 흐름이 직관적

## 테스트 시나리오 3: 콜 그래프 탐색

**목표**: 특정 메소드의 호출 관계를 시각적으로 탐색할 수 있는가?

### Task 흐름
1. "콜 그래프" 탭 클릭
2. 메소드 노드 클릭으로 그래프 확대
3. 깊이 슬라이더로 탐색 깊이 조절
4. 노드 클릭으로 메소드 상세 모달 열기

### 결과

| 단계 | 상태 | 메모 |
|------|------|------|
| 콜 그래프 탭 전환 | ✅ | 즉시 React Flow 렌더링 |
| dagre 자동 레이아웃 | ✅ | 계층형 위→아래 방향으로 정돈 |
| 노드 색상 구분 | ✅ | 파란(사용)/빨간(미사용) 색상 구분 |
| 줌/팬 조작 | ✅ | 마우스 휠/드래그 자연스럽게 동작 |
| 깊이 슬라이더 (1-3) | ✅ | 슬라이드 시 그래프 즉시 재렌더링 |
| 노드 클릭 → 모달 | ✅ | 메소드 상세 모달 팝업 |
| 전체화면 버튼 | ✅ | 그래프 영역 확대 |

**완료 시간**: 약 20초 (탭 전환 + 조작 탐색)
**클릭 수**: 4-6회 (탭 + 노드 + 슬라이더 + 모달 닫기)

### 발견된 이슈
- **경미**: 노드 수 많을 때 초기 줌 레벨이 너무 작음 → "Fit View" 버튼으로 해소
- **없음**: 주요 UX 장애 없음
