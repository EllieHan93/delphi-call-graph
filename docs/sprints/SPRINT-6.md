## 의사결정 기록 (Decision Records)

### DR-6.1: 인라인 그래프 탭 vs 모달

**결정**: Call Graph를 별도 탭 내 전체 뷰로 배치 (Sprint 5의 모달 방식에서 변경)

**이유**:
- 모달은 화면의 80%를 차지해도 그래프 탐색 공간이 부족
- 탭 전환으로 그래프를 전체 화면으로 사용 → depth 슬라이더 조작이 편리
- "그래프 보기" 버튼 클릭 → Call Graph 탭 자동 전환으로 UX 연속성 유지

### DR-6.2: 가로 스택 바 차트 직접 구현

**결정**: Chart.js/Recharts 라이브러리 대신 Tailwind CSS div 기반 스택 바 차트 구현

**이유**:
- Chart.js 번들 ~200KB 추가 vs 커스텀 div 0KB
- 인터랙션(클릭 → 테이블 필터 연동)을 직접 제어 가능
- 디자인 토큰(Success/Danger 색상) 직접 적용

## 회고 (Retrospective)

### 잘된 점
- ARIA 탭 패턴 완전 구현으로 키보드 전용 네비게이션 가능
- 스켈레톤 UI가 분석 대기 시간의 인지적 부담 감소 효과

### 개선할 점
- 탭 구조 결정이 Sprint 4에서 이미 되었어야 했음 (Sprint 6에서 리팩터링 비용 발생)
- UnitChart 바 차트 hover 툴팁을 CSS only로 구현 → 복잡한 내용 표시에 한계

### 예상 vs 실제 소요 시간
- 예상: 마무리 + E2E 4시간
- 실제: 6시간 (탭 리팩터링 + E2E 테스트 환경 설정)
