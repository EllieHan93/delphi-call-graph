## 의사결정 기록 (Decision Records)

### DR-4.1: TanStack Table 선택

**결정**: 네이티브 `<table>` 또는 ag-Grid 대신 TanStack Table v8 채택

**이유**:
- Headless 아키텍처 → Tailwind CSS 디자인 토큰과 완전한 통합
- 정렬/필터/페이지네이션을 별도 상태 관리 없이 hook으로 처리
- ag-Grid 등 완성형 라이브러리 대비 번들 크기 작음
- 접근성(`aria-sort`, `scope="col"`) 직접 제어 가능

### DR-4.2: 슬라이드오버 vs 모달 (메소드 상세)

**결정**: 메소드 테이블 상세에는 슬라이드오버(오른쪽에서 진입), 콜 그래프에서는 모달

**이유**:
- 슬라이드오버: 테이블 맥락을 유지하면서 상세 정보 확인 가능 (UX 연속성)
- 모달: 그래프 위에 오버레이로 표시 → 그래프 전환 없이 상세 확인

## 회고 (Retrospective)

### 잘된 점
- Tailwind `extend`에 디자인 토큰 등록으로 전체 컴포넌트 일관성 확보
- `aria-live="polite"` + `role="alert"` 조합으로 스크린 리더 지원 처음부터 설계

### 개선할 점
- TanStack Table의 서버사이드 필터링 연동이 예상보다 복잡 (클라이언트 필터로 우선 구현)
- 상태 배지 색상이 PRD 토큰과 처음에 불일치 → 수정 필요

### 예상 vs 실제 소요 시간
- 예상: 대시보드 UI 6시간
- 실제: 8시간 (TanStack Table 학습 + 디자인 토큰 정합성 조율)
